# 智咖系统 - HA集成

Home Assistant 自定义集成，与智咖系统服务端配合使用，提供设备在线检测和WOL远程唤醒功能。

## 功能

- **设备同步** - 自动从智咖系统服务端获取设备列表
- **WOL唤醒** - 远程唤醒关机的电脑
- **在线检测** - 定时Ping检测设备在线状态
- **REST API** - 提供设备状态查询接口

## 安装方法

### 方法一：手动安装

1. 下载本目录 `smartcafe_control` 中的所有文件
2. 复制到 Home Assistant 的 `custom_components/` 目录：
   ```
   <HA配置目录>/custom_components/smartcafe_control/
   ```
3. 重启 Home Assistant
4. 进入 **设置** → **设备与服务** → **添加集成**
5. 搜索 **智咖系统** 并添加
6. 填写配置：
   - **服务端地址**: 智咖系统服务端地址（如 `http://192.168.1.100:8765`）
   - **长期访问令牌**: HA的长期访问令牌（在HA个人资料页面创建）

### 方法二：通过HACS安装

1. 打开 HACS → 自定义存储库
2. 添加仓库地址：`https://github.com/zhutianlei/SmartCafe-OS`
3. 选择类别为 **集成**
4. 安装 **智咖系统**
5. 重启 Home Assistant
6. 在集成中添加并配置

## 配置说明

| 配置项 | 说明 |
|--------|------|
| 服务端地址 | 智咖系统服务端的HTTP地址 |
| 长期访问令牌 | 用于访问HA API的令牌 |

### 获取长期访问令牌

1. 登录 Home Assistant
2. 点击左下角用户头像 → **个人资料**
3. 滚动到页面底部 → **长期访问令牌**
4. 点击 **创建令牌**
5. 输入名称（如 "智咖系统"）并复制生成的令牌

## 安装后的实体

安装成功后，每个设备会自动创建以下实体：

| 实体类型 | 说明 |
|----------|------|
| Switch | 电脑电源开关，点击发送WOL唤醒信号 |
| BinarySensor | 在线状态传感器，显示设备是否在线 |

## 使用方法

1. 确保智咖系统服务端已安装并运行
2. 在服务端管理面板中添加设备（需要填写MAC地址）
3. 安装本集成并配置服务端地址
4. 集成会自动同步设备列表
5. 在HA仪表板中添加Switch和BinarySensor实体

## REST API

集成提供以下API供外部调用：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/pc_manager/devices` | GET | 获取所有设备列表 |
| `/api/pc_manager/status` | 获取所有设备在线状态 |
| `/api/pc_manager/devices/{ip}/wake` | POST | 唤醒指定设备 |

## 故障排除

**设备状态不更新？**
- 检查服务端地址是否正确
- 确认长期访问令牌有效
- 查看HA日志中的错误信息

**WOL唤醒不工作？**
- 确保设备已配置MAC地址
- 确保设备支持WOL功能
- 确保设备已连接电源并开启了WOL
